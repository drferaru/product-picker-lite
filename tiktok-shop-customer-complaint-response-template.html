Copyable template

Damaged, defective, or not-as-described response

Keep the tone calm. Do not argue with the buyer. Capture the proof you need, then close the loop through the platform path.

Hi [Name], thanks for letting us know. I’m sorry the item arrived this way. To fix it quickly, please send one clear photo showing [damage / wrong variant / missing part] and one photo of the package label if available. Once we confirm the issue, we can help with the correct platform-supported resolution: [replacement / refund / missing part support]. I’m also logging this as a product quality issue so we can check whether this is a one-off shipment problem or something we need to pause before restocking.

Operator note: if the complaint repeats twice on the same SKU, stop treating it as support noise. It is now a product validation signal.

Supplier escalation

When support tickets become supplier proof

After two similar complaints, send a concise supplier message. Do not just ask for a discount. Ask for the exact proof that prevents the next refund.

Hi [Supplier], we received repeated customer complaints about [specific issue] on the sample/test units. Before we place any reorder, please send: 1. A photo of the current packaging for this SKU. 2. A photo showing the variant/parts check before shipment. 3. Your defect replacement policy for this exact issue. 4. What changed operationally so this does not repeat. If we cannot verify this, we will pause the SKU and test a different supplier/product.

Reject signal: vague replies like “do not worry,” “quality is good,” or “we will check next time” are not enough for reorder money.
